About This Project

This project was built as part of the CodSoft Machine Learning Internship. The goal was to apply end-to-end machine learning — from raw data exploration all the way to a deployed prediction API — on the classic Titanic passenger dataset.

The dataset contains records of 891 passengers aboard the RMS Titanic, which sank on April 15, 1912. The challenge: predict whether a given passenger survived based on their demographic and ticketing information. 38% of passengers survived the disaster.

Problems Found & Solved
  • Missing Age values — 177 out of 891 passengers had no age recorded. Solved by filling with the column mean, then casting to integer to keep the dtype consistent.
  • Missing Embarked values — 2 rows had no port of embarkation. Since it was negligible, those rows were dropped with dropna.
  • Irrelevant columns — PassengerId, Name, Ticket, and Cabin had no predictive value and added noise. All four were dropped before modelling.
  • Categorical encoding — Sex (male/female) was label-encoded to 0/1. Embarked (S/C/Q) was ordinally mapped to 2/0/1 based on frequency.
  • Skewed Fare distribution — Fare was heavily right-skewed with many outliers. A log1p transform was applied inside the sklearn Pipeline to normalise the distribution without touching the raw data.
  • Feature engineering — SibSp (siblings/spouses aboard) and Parch (parents/children aboard) were merged into a single FamilySize feature (SibSp + Parch + 1). This reduced dimensionality and captured the insight that solo travellers and very large families had notably lower survival rates.

XGBoost was selected as the final production model for its robustness to overfitting and strong generalisation. The trained pipeline was serialised with pickle and served via a Flask REST API.

Features Used
  • Pclass — Ticket class (1, 2, 3). Strong proxy for socioeconomic status and cabin deck location.
  • Sex — "Women and children first" was enforced during evacuation, making sex the single strongest predictor.
  • Age — Children had higher priority. Elderly passengers had lower survival rates due to mobility.
  • Fare — Log-transformed. Highly correlated with class and therefore deck/lifeboat proximity.
  • Embarked — Port of boarding. Correlated with passenger class mix on that boarding group.
  • FamilySize — Engineered. Passengers travelling alone or in groups of 5+ had the lowest survival rates.
Pipeline Architecture

All preprocessing lives inside an sklearn Pipeline with a ColumnTransformer that applies log1p only to the Fare column, passing all other features through unchanged (remainder='passthrough'). This prevents data leakage and means the full model — preprocessing + classifier — is packaged in one .pkl file ready for inference.
